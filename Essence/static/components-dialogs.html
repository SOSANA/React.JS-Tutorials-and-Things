Dialogs

As you might expect, dialog boxes pop. And they have actions - set the dialog content and its actions with .e-dialogs-content and .e-dialogs-actions, as well as the button type & text. Thatâ€™s it.

Dialog title

When text labels exceed the maximum button width, use stacked buttons to accommodate the text. Affirmative actions are stacked above dismissive actions.

REACT CODE

          
<Btn>
  <BtnItem
    type='primary'
    classes={'raised'}
    label='Show Simple Dialog'
    rippleEffect={true}
    actionClick='dialog'
    actionType={{
      'action': 'show',
      'id': 'dialog-simple'
    }}
  />
</Btn>

<Dialog>
  <DialogItem id='dialog-simple'>
    <DialogItemHeader>
      <Text type='h2'>Dialog title</Text>
    </DialogItemHeader>

    <DialogItemContent>
      <Text type='p'>
        When text labels exceed the maximum button width,
        use stacked buttons to accommodate the text.
        Affirmative actions are stacked above dismissive actions.
      </Text>
    </DialogItemContent>

    <DialogItemFooter>
      <Btn>
        <BtnItem
          type='danger'
          classes={'flat'}
          label='Disagree'
          actionClick='dialog'
          actionType={{
            'action': 'hide',
            'id': 'dialog-simple'
          }}
        />
        <BtnItem
          type='primary'
          classes={'flat'}
          label='Agree'
          actionClick='dialog'
          actionType={{
            'action': 'hide',
            'id': 'dialog-simple'
          }}
        />
      </Btn>
    </DialogItemFooter>
  </DialogItem>
</Dialog>

How to use

Use the "Dialog" Essence component to render a group or single "DialogItem" components. The "Dialog" Essence component will return a div container with a group or single <div> html tag with the class name "e-dialog".

• Structure 

<Dialog>
  <DialogItem>
    <DialogItemHeader>
      Header: can be used to include Image and/or Text
    </DialogItemHeader>
    <DialogItemContent>
      Content: can be used to include any component
    </DialogItemContent>
    <DialogItemFooter>
      Footer: used for actions from single or multiple BtnItem items
    </DialogItemFooter>
  </DialogItem>
</Dialog>
• Styling
- classes: add extra class names to any Dialog component or subcomponent from Styles > Colors menu

• Options
- id: required for unique DialogItem to be shown - use the examples from above to structure your own Dialog

How to use - Essence Native

Use "DialogItem" to render native iOS "AlertIOS" component with "title" and "message" options.

• Structure 

<DialogItem 
  title={'Dialog title'} 
  message={'Dialog message'}
  > 
  {[
    {
      text: 'OK', 
      onPress: () => customOKFunction()
    },
    {
      text: 'Cancel',
      onPress: () => customCancelFunction()
    }
  ]}
</DialogItem>
• Styling
- no styling supported

• Options
- title: required for AlertIOS "title" option
- message: optional for AlertIOS "message" option

• Content
- array format: "text" as string and "onPress" as function
