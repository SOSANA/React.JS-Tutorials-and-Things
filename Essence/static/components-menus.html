Menus

Menus allow users to take an action by selecting from a list of choices revealed upon opening a temporary, new sheet of material. Menus in material design can be simple, covering the first item, and cascading. When working for a mobile environment use the .mobile class to make things more visible.

How to use

Use the "Menu" tag to render a group of "MenuItem" components. The "Menu" Essence component will return a nav container with class name "e-nav-menu". Inside the Menu component you can include only the MenuItem component.
A "MenuItem" Essence component will render any component or html code.

• Structure 

<Menu id='menu-cover'
  classes={'cover mobile'}
  placeholder='Show Menu Cover'
>
  <MenuItem>
    can be used to include any component or html code
  </MenuItem>
</Menu>
• Styling
- classes: add extra class names to any Menu component or subcomponent from Styles > Colors menu


• Options
- id: required for unique Menu to be shown on click
- placeholder: is the text to be shown at the render starting point
- use the examples from above to structure your own Menu

How to use - Essence Native

Use the "Menu" Essence component to render a group of "MenuItem" components. The "Menu" Essence component will return a nav container with class name "e-nav-menu". Inside the Menu component you can include only the MenuItem component.
A "MenuItem" Essence component will render any component or html code.

• Structure 

<Menu
  placeholder='Select option'
  textStyle={__ColorsStyles__}
>
  {[
    'Menu option 1',
    'Menu option 2',
    'Menu option 3',
    'Menu option 4',
  ]}
</Menu>
• Styling
- textStyle: add styling class names to text Menu children from Styles > Colors menu


• Options
- placeholder: is the text to be shown at the render starting point, on Menu option selection will be replaced with the option value or text

• Content
- array format: children type as text string
