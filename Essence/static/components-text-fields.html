How to use

Use the "Input" Essence component to render a "InputItem" component. The "Input" Essence component will return a div container with class name "snackbar". Inside the Input component you can include only the InputItem component.

• Structure 

<Input>
  <InputItem
    classes={'e-input-group has-warning'}
    inputClasses={'e-input e-input-success empty'}
    type='text'
    name='label'
    label='Did you know each time you see a full moon you always see the same side'
  />
</Input>
• Styling
- classes: add extra class names to Input component div container from Styles > Colors menu
- inputClasses: add extra class names to InputItem component input or textarea container from Styles > Colors menu


• Options
- id: optional to set the id attribute for the InputItem tag
- type: default is text, other type: textarea, email, search, number
- name: required to set the Input name attribute
- label: optional to set the Input placeholder attribute
- placeholder: optional to set the Input placeholder attribute
- disabled: if set to true it uses the "disabled" input attribute
- counter: if set to an integer value will set the maximum limit of characters
- hint: a text explaining what this input is about
- actionClick: any value from the following list: navigation, bottomsheets, datepicker or dialog
- actionType: has the following options:
-   action: 'setValue' to set the value from a remote component
-   id: any string that represent the 'id' option for a component
- use the examples from above to structure your own Text field

How to use - Essence Native

Use the "InputItem" Essence component to render iOS React Native TextInput component.

• Structure 

<InputItem
  style={__ColorsStyles__}
  value='text input value'
  editable={false}
  placeholder='placeholder text input'
  multiline={true}
  password={true}
/>
• Styling
- style: add styling class names to InputItem component from Styles > Colors menu - will style the text input
- placeholderTextColor: add styling class names the placeholder string from Styles > Colors menu


• Options
- value: default value for the text input
- editable: if set to false text is not editable
- placeholder: string that will be rendered before text input has been entered
- multiline: if set to true text input will render multiple lines as a textarea html
- clearButtonMode: when the clear button should appear on the right side of the text view using one value from the list: 'never', 'while-editing', 'unless-editing', 'always'
- password: if set to true text input obscures the text entered so that sensitive text like passwords stay secure
- keyboardType: determines which keyboard to open from the list: 'default', 'ascii-capable', 'numbers-and-punctuation', 'url', 'number-pad', 'phone-pad', 'name-phone-pad', 'email-address', 'decimal-pad', 'twitter', 'web-search', 'numeric'
- for other options please see the TextInput React Native component
